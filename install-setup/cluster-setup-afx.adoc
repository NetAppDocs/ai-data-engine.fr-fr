= Configurer et paramétrer AI Data Engine
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
En tant qu'administrateur de stockage ONTAP, vous utilisez ONTAP System Manager pour intégrer AI Data Engine (AIDE) à votre environnement AFX. Après avoir terminé l'installation de vos nœuds de calcul de données, ONTAP peut les découvrir et les ajouter à un cluster. Vous pouvez ajouter des nœuds AIDE lors de la configuration initiale du cluster AFX ou les ajouter à un cluster AFX existant, puis effectuer la configuration AIDE requise.

.Avant de commencer
* Vous avez des NetApp DCN ou des serveurs tiers installés et connectés aux commutateurs du cluster. Consultez la documentation d'installation pour link:../install-aide-server/install-workflow.html["AI Data Engine software sur serveurs tiers"] ou link:../install-dcn/install-workflow.html["DCN NetApp"] pour plus d'informations.
* (Pour les serveurs tiers) Vous avez effectué link:../install-setup/install-aide-as-software.html["Installation du logiciel AIDE"] sur chaque serveur. ONTAP peut alors détecter ces serveurs comme des nœuds de calcul, et vous pouvez les ajouter à un cluster.




== Configurez AIDE et intégrez-le à un système de stockage AFX

Il existe deux façons de configurer AIDE et de l'intégrer à un système de stockage AFX. Choisissez l'option qui convient à votre environnement.



=== Configurez AIDE avec un nouveau cluster AFX

Si vous configurez AIDE dans le cadre du déploiement initial d'un système AFX, le processus de configuration d'AIDE fait partie du flux de travail standard de configuration du cluster AFX. Lors de la configuration du cluster ONTAP, System Manager détecte automatiquement les nœuds de calcul connectés et les intègre à la configuration du cluster.


NOTE: Vous devrez link:../install-setup/configure-oidc.html["configurer l'authentification OpenID Connect"] pour activer l'accès à AIDE Console.



=== Configurez AIDE avec un cluster AFX existant

Si vous ajoutez des nœuds de calcul (soit des NetApp DCN, soit des serveurs tiers) à un système de stockage AFX existant, vous devez ajouter les nœuds et terminer la configuration de votre cluster AIDE.



==== Ajouter des nœuds de calcul au cluster

ONTAP détecte les nœuds de calcul connectés à votre réseau de cluster et les affiche dans le Gestionnaire système. link:../administer/dcn-add-node.html["Ajoutez des nœuds de calcul à votre cluster de stockage pour en faire un cluster AIDE"].



==== Terminez la configuration de votre AIDE cluster

Après avoir ajouté les nœuds de calcul et terminé la configuration réseau initiale, vous pouvez finaliser la configuration de votre nouveau AIDE cluster.

.À propos de cette tâche
Une fenêtre d'aide à la configuration s'affiche sur la page avec les étapes de configuration nécessaires. Une coche apparaît à gauche de chaque étape ou action déjà effectuée. Une flèche (image:icon_arrow.gif["Flèche bleue pointant vers la droite"]) s'affiche pour les éléments qui n'ont pas encore été terminés. Pour chaque action incomplète :

* Sélectionnez le lien du titre activé et complétez l'action de configuration pour votre environnement.
* Si le lien est désactivé, survolez le titre pour afficher l'action requise pour l'activer et continuez avec l'action de configuration.


Chaque étape comporte un lien vers une documentation supplémentaire lorsque cela est possible. Une partie de la configuration peut être incluse dans l'installation du cluster AFX https://docs.netapp.com/us-en/ontap-afx/["Documentation AFX"^].

.Étapes
. Sélectionnez *Configurer les groupes d’agrégation de liens et les VLAN*.
+
Si votre environnement l'exige, vous pouvez configurer des LAG et des VLAN au niveau du cluster. La sélection de cette option ouvre la page des ports Ethernet.

. Sélectionnez *Configurer les protocoles réseau*.
+
Vous pouvez configurer les protocoles réseau de la machine virtuelle de stockage (SVM).

. Sélectionnez *Update data compute software*.
+
Si une image logicielle AIDE est disponible pour une mise à jour, cette option sera activée. Sélectionnez-la pour accéder à la page de mise à jour du logiciel. Pour plus d'informations, consultez link:../administer/update-aide-firmware-software.html["Mettre à jour le logiciel AIDE"].

. Sélectionnez *Configurer la mise en réseau de l'AI Data Engine*.
+
Cette option affiche la page permettant de configurer l'adresse réseau du data engine.

. Sélectionnez *Créer des interfaces réseau intercluster*
+
Cette option ouvre la page pour configurer les interfaces réseau inter-clusters dans le cadre d'une procédure en deux étapes si vous souhaitez cataloguer des données distantes.

. Sélectionnez *Peer avec d'autres systèmes de stockage ONTAP*.
+
Après avoir créé les interfaces réseau inter-clusters, vous pouvez établir des relations de peering avec d'autres systèmes de stockage ONTAP.

. Sélectionnez *Ajouter un container de données*.
+
Vous pouvez ajouter un volume destiné à être utilisé par AIDE et l'associer à une SVM spécifique.

. Sélectionnez *Add Workspaces*.
+
Cela ouvre la page des espaces de travail de Data Engine où vous pouvez link:../workspaces/create-workspace.html["créer un espace de travail"].

. Sélectionnez *Configure OpenID Connect*.
+
Vous devrez link:../install-setup/configure-oidc.html["configurer l'authentification OpenID"] pour activer l'accès à AIDE Console.



.Après avoir terminé
Une fois toutes les actions terminées, la fenêtre d'aide est masquée. Vous pouvez la rouvrir et la fermer manuellement selon vos besoins.



== Activez votre licence AIDE pour les déploiements NetApp DCN

Si votre configuration inclut des NetApp DCN, vous devez link:../install-setup/install-license-task.html["Installez votre licence de services premium AIDE"] après que votre cluster AIDE est installé et intégré au système de stockage AFX. La licence des services premium AIDE est requise pour bénéficier de la fonctionnalité complète d'AIDE, y compris la vectorisation et les capacités de garde-fous.
